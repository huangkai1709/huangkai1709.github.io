git的作用

  • 分布式版本控制
  • 多开发人员协调工作
  • 有效监听谁做的修改
  • 本地及远程操作

git配置

  • git config –global user.name ”user”
  • git config –global user.emal ”xxx@xxx.com“

git指令

  1. git init //初始化本地仓库

  2. git add //添加文件

  3. git rm –cached //删除文件
  4. git status //查看状态
    • 本地仓库修改时status会改变
  5. git commit //提交
    • 添加提交信息 :wq退出
    • git commit -m ‘changed index.html’ //提交并且备注
  6. git push //推送到仓库
  7. git pull //从远程仓库拉取数据
  8. git clone 仓库地址 //从远程仓库拷贝数据
  9. touch .gitignore //创建文件，创建忽略文件
    • 在.gitignore中添加需要忽略的文件路径
    • 文件log.txt
    • 文件夹/dir1
  10. git branch xxx //创建分支
    • git checkout xxx //进入xxx分支
    • git checkout master //回到主线
    • 主线与分支不会相互影响
  11. git merge xxx //在master使用该指令能够合并xxx分支
  12. git remote add origin 仓库地址 //与仓库对接
